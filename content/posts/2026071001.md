### 2. 高并发：每增加一个线程，价格标签就翻一次

企业场景最怕的是什么？并发。同一个时间点，1000个用户在疯狂提问。

如果你不做并发控制，Java的 `ExecutorService` 会瞬间产生成百上千个HTTP请求。[千聚ai中转站](https://www.qianjuai.com/)虽然支持**无限制并发**， 但你的钱包不支持。

**【代码示例】**

java

ExecutorService executor = Executors.newFixedThreadPool(100);

for (int i = 0; i < 1000; i++) {

        // 直接调用API，完全不控制
        // 这会瞬间消耗大量Token

**价格标签：** 假设一次对话平均消耗2000个Token（输入输出混合），1000次并发就是2,000,000 Token。按DeepSeek-R1的0.6倍费率，输入约$2.4，输出约$9.6——总共**十几块钱**眨眼就没了。如果你不做频率限制，老板可能会在下班前找你谈话。

正确的做法是引入信号量或滑动窗口。

java
// 这是最划算的代码块
Semaphore semaphore = new Semaphore(50); // 限制最大50个并发

for (int i = 0; i < 1000; i++) {
    executor.submit(() -> {
        if (semaphore.tryAcquire()) {
            try {
                // 调用API，速度控制在合理范围内
            } finally {
                semaphore.release();
            }
        }
    });
}


**价格标签：** 并发从100降到50，成本瞬间减半，同时用户体验不会受到丝毫影响。这一行 `Semaphore` 代码，是你给企业省下的第一笔真金白银。
