## 先聊一个扎心的“官方Python示例”

不少开发者在GitHub或官方文档上，看到类似这样的GLM-4.5 Python示例代码：

python

from openai import OpenAI

client = OpenAI(
    api\_key="your\_glm\_api\_key",
    base\_url="https://open.bigmodel.cn/api/paas/v4"
)
response = client.chat.completions.create(
    model="glm-4v-plus",
    messages=\[
        {"role": "user", "content": "请总结这个文档"}
    ],
    temperature=0.7,
    max\_tokens=2000
)
print(response.choices\[0\].message.content)

这段代码看起来人畜无害，对吧？实际上，它至少埋了三个“坑”，让很多新人在不知不觉中花掉冤枉钱。

**第一个坑：`max_tokens` 参数设得太高。** 官方默认值往往偏大，如果你不做控制，每次对话都可能输出远超实际需要的字数，Token消耗直接翻倍。

**第二个坑：`model` 字段的版本陷阱。** “glm-4v-plus”并不是一个固定的模型名称，它可能是某个高倍率计费的商业版名称，与普通版的API价格完全不同。文档里一笔带过，账单上却是天壤之别。

**第三个坑：历史消息缓存不设置。** 默认情况下，系统会把整个对话历史都塞进每一次请求里，对话轮次一多，每次的Token消耗指数级增长。

这三个坑单拎出来一个都还好，但合在一起，就是妥妥的“烧钱三件套”。对于个人开发者或小型团队来说，这种开销完全没必要。
