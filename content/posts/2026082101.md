### 第二步：编写核心调用代码

java
import com.qianjuai.unet.Entity.AzureGPT5Pro;
import com.qianjuai.unet.Entity.AWSBedrockPro;
import com.qianjuai.unet.Entity.GoogleVertexAIPro;
import com.qianjuai.unet.manager.UnifiedManager;

public class QuickStart {
    public static void main(String[] args) {
        // 初始化UnifiedManager，无需管道配置，无需注册任何环境变量
        UnifiedManager manager = new UnifiedManager(
            "https://www.qianjuai.com/v1", // API Base
            "your-session-key-from-free-subsite" // 从免费子站拿到的临时Key
        );

        // 场景1：同时用Azure GPT-5 Pro和Google Gemini 2.5 Pro做答案对比
        AzureGPT5Pro azureModel = new AzureGPT5Pro("gpt-4o", "low-latency");
        GoogleVertexAIPro googleModel = new GoogleVertexAIPro("gemini-2.5-pro", "balanced");

        // 低代码调用：执行方法，自动路由
        String azureResponse = manager.generate(azureModel, "用中文解释一下Java多态");
        String googleResponse = manager.generate(googleModel, "用中文解释一下Java多态");

        System.out.println("Azure回复: " + azureResponse);
        System.out.println("Google回复: " + googleResponse);

        // 场景2：自动故障转移——当A平台失败，自动B平台，无需if-else
        manager.withFailover(
            new AWSBedrockPro("claude-3-5-sonnet", "us-east-1")
        ).generate("写一段生产级的JPA分页查询代码");
    }
}


**注意**：在上述代码中，`model`对象就像实体一样，你可以定义它的`temperature`、`max_tokens`等属性，所有底层映射全部由千聚力平台处理。**无需任何注册流程**，拿到Session Key即刻运行。

### 第三步：通过配置文件实现一键切换

在Java项目中，如果你更倾向于配置方式管理，可以在`application.properties`里这么定义：

properties
# 默认路由：自动切换至延迟最低的模型
unet.default.entity=com.qianjuai.unet.Entity.AzureGPT5Pro
unet.default.model=gpt-4o
unet.failover.order=AWS,GCP,Azure

# 低代码模式：直接“一键”新增第三个平台，无需改任何代码
unet.platforms=azure,aws,gcp


把代码交给CI/CD，当你添加第四个平台（比如阿里云通义千问）时，只需要在`properties`里增加一行`unet.platforms.add=aliyun`即可，**实体自动继承协议**，让你真正打破常规。
