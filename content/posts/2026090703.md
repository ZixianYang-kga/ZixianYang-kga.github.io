## Node.js接入示例：真的就是5分钟的事情

技术说再多，不如直接上代码。下面这个Node.js示例，将展示如何用不到30行代码，实现对智谱清言和OpenAI双模型的并行调用。

javascript
// 1. 安装依赖：确保你的项目里有 openai 这个包
// npm install openai

import OpenAI from 'openai';

// 2. 创建统一客户端 —— 这就是整个工程的核心，没有第二条连接
const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1', // 关键！指向统一的智谱清言国内接入点
  apiKey: 'your-api-key' // 替换为你从智谱清言国内平台申请的Key
});

// 3. 定义一个通用的调用函数，模型名称通过参数动态传入
async function callModel(modelName, userMessage) {
  const chatCompletion = await client.chat.completions.create({
    model: modelName, // 这里可以传入任何支持的模型名，如 'qianfan_chinese_model', 'gpt-4o-mini'
    messages: [{ role: 'user', content: userMessage }],
  });
  return chatCompletion.choices[0]?.message?.content;
}

// 4. 开始调度：5分钟内你就能跑通的示例
async function main() {
  console.log('正在询问智谱清言...');
  const qianfanResponse = await callModel(‘qianfan_chinese_model’, ‘用一句话解释人工智能。’);
  console.log('智谱清言回复:', qianfanResponse);

  console.log('正在询问OpenAI...');
  const openaiResponse = await callModel('gpt-4o-mini', '用一句话解释人工智能。');
  console.log('OpenAI回复:', openaiResponse);

  console.log('多模型调度完成！');
}

main();


看到了吗？你甚至不需要知道对方模型的服务器在哪，不需要写任何 if-else 的路由选择逻辑。你只需要在 `model` 参数里改一个名字，就能在智谱清言和OpenAI之间随意“换台”。整个代码的维护成本，几乎为零。
