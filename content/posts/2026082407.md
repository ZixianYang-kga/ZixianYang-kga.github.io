## 第二步：直连调通，代码里的“一行改动”

拿到Key，是不是就大功告成？

还没完。很多人下一步就是把Key一股脑塞进代码，然后直接调用 `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent` 这个官方地址。

**这里有第二个大坑：** 你的服务器或者本地环境，如果无法直连 `googleapis.com`，这个请求就会超时报错。很多人在这里一卡就是半天，以为是Key无效，其实是网络问题。

**避坑点：修改Base URL，一步到位**

解决方法极其简单：把请求的 `base_url`（基础地址）从Google官方地址，改成中转平台的地址即可。

千聚AI中转站的API接口地址是：`https://www.qianjuai.com/v1`

它完全兼容OpenAI的接口格式。也就是说，你以前写好的任何调用OpenAI GPT的代码，只需要改 **两个地方**：

1.  `base_url` 改成 `https://www.qianjuai.com/v1`
2.  `api_key` 改成你在千聚后台拿到的Key

**来看一个最典型的调用示例（Python + OpenAI库）：**

python
from openai import OpenAI

# 原来的代码可能长这样：
# client = OpenAI(api_key="你的OpenAI_KEY", base_url="https://api.openai.com/v1")

# 现在改成这样：

    api_key="sk-your-qianju-api-key",  # 替换成你在千聚生成的Key

    model="gemini-2.5-flash", # 注意，模型名称要写千聚支持的版本
    messages=[
        {"role": "user", "content": "你好，请用中文简单介绍一下自己。"}
    ]
)

print(response.choices[0].message.content)


就这么简单。
