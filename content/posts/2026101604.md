## 第二步：配置DNS解析

域名买好之后，需要把它指向你的服务器IP。

**操作步骤：**
1. 登录域名注册商的控制台，找到“DNS解析”或“域名管理”。
2. 添加一条“A记录”，主机记录写 `@` 或 `api`（看你的需要），记录值填你的服务器公网IP。
3. 添加一条“CNAME记录”，把 `www` 指向你的主域名（可选，用于跳转）。

**我的配置示例：**

| 记录类型 | 主机记录 | 记录值 | TTL |
|----------|----------|--------|-----|
| A        | @        | 123.123.123.123 | 600 |
| A        | api      | 123.123.123.123 | 600 |
| CNAME    | www      | yourdomain.com | 600 |

**注意点：** DNS解析有缓存，配置后可能要等1-2小时才能生效。建议用 `ping yourdomain.com` 验证一下，等结果返回正确IP再继续。

## 第三步：搭建Nginx反向代理服务器

这是核心环节，我会尽量写得细一些。

**服务器准备：**
我选择的是阿里云轻量应用服务器（香港节点），系统镜像 `Ubuntu 20.04`，配置：2核4G，带宽 30Mbps。如果你预算有限，用国内服务器也可以，但需要确保有备案。

**安装环境：**

bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装Nginx
sudo apt install nginx -y

# 安装SSL证书工具
sudo apt install certbot python3-certbot-nginx -y


**配置Nginx站点：**

bash
# 创建站点配置文件
sudo nano /etc/nginx/sites-available/api.yourdomain.com

# 写入以下内容
server {
    listen 80;
    server_name api.yourdomain.com;

    location /v1/ {
        proxy_pass https://api.openai.com/v1/;
        proxy_set_header Host api.openai.com;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        # 流式输出配置
        proxy_buffering off;
    }
}

# 创建软链接并启用站点
sudo ln -s /etc/nginx/sites-available/api.yourdomain.com /etc/nginx/sites-enabled/
sudo nginx -t  # 测试配置是否正常
sudo systemctl reload nginx


**关键点解释：**
- `proxy_pass` 是反向代理的核心，把请求转发到OpenAI官方API。
- `proxy_buffering off` 是为了支持流式输出，不然用GPT-4的时候会卡住。
- `proxy_set_header` 确保请求头信息准确传递。

## 第六步：调试接口（验明正身）

这是最后一步，也是最关键的：验证你的中转站能不能被正常调用。

**测试环境：**
假设你的域名为 `api.yourdomain.com`，One API安装在 `yourdomain.com:3000`。

**测试脚本（Python）：**

python
import openai

openai.api_base = "https://api.yourdomain.com/v1"  # 这里改成你的一级域名
openai.api_key = "你的客户端Key"

try:
    response = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[
            {"role": "user", "content": "你好，请用中文回复"}
        ],
        stream=True  # 测试流式输出
    )
    
    for chunk in response:
        if chunk['choices'][0]['delta'].get('content'):
            print(chunk['choices'][0]['delta']['content'], end='')
except Exception as e:
    print(f"请求失败：{e}")


**成功标准：**
- 非流式输出能够正常返回完整响应。
- 流式输出能够逐字打印，没有超时或卡顿。
- 速度在可接受范围内（国内直连一般延时在500ms以内）。

**常见问题排查：**
- **报错 401**：API key填错或未设置。
- **报错 403**：域名未备案或者被防火墙拦截。
- **报错 502**：Nginx或One API服务未启动，或者端口配置不正确。
- **流式输出失败**：确认Nginx中已关闭 `proxy_buffering`。

## 第七步：性能优化与安全加固

一次跑通只是基础，为了让中转站稳定运行，还需要做一些优化。

**性能优化：**
- **启用缓存**：对相同的请求做缓存，减少对OpenAI API的重复调用（适合提示词固定的场景）。
- **限速限流**：防止被滥用。One API自带限流功能，可以设置每分钟每个用户的请求数。
- **CDN分发**：如果是面向全国用户，考虑使用国内CDN加速静态资源和部分动态请求。

**安全加固：**
- **IP白名单**：只允许信任的IP访问管理后台。
- **请求日志审计**：记录用户的API调用记录，便于排查滥用。
- **密钥轮换**：定期更换客户端Key和服务端Key。
