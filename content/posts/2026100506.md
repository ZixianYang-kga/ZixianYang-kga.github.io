### 流式输出（Streaming）—— 像ChatGPT一样实时显示结果

如果你的应用场景需要面对用户，比如你正在开发一个聊天机器人，那么流式输出（Streaming）就非常重要。它能实时地返回模型生成的文字，而不是等全部生成完毕才一次性输出，大大提升了交互体验。

**流式调用的完整代码：**

python
from openai import OpenAI

client = OpenAI(
    api_key="你的千聚API密钥",
    base_url="https://www.qianjuai.com/v1"
)

# 与普通调用唯一的区别：加入 stream=True 参数
stream = client.chat.completions.create(
    model="grok-4-fast",
    messages=[
        {"role": "user", "content": "请给我讲一个关于未来AI技术的小故事，100字以内。"}
    ],
    stream=True,  # 这里开启流式输出
    max_tokens=2048
)

# 逐块遍历流式数据
for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="")


在终端运行这段代码，你会看到Grok-4的回复像打字机一样，一个字一个字地蹦出来，完全没有延迟感。很多开发的实时AI应用（比如AutoGPT、个人助手）都基于这个模式搭建。

## 常见错误排查（小白必看）

我相信大多数读者跟着上面的步骤都能一次跑通，但万一遇到报错，别慌，90%的错误都在下面这两个问题里：

**错误1: `ConnectionError: HTTPSConnectionPool` 或 `TimeoutError`**
- **原因：** 你的网络环境没有直连到服务器，还是走了默认的OpenAI官方域名。
- **解决：** 百分之百确认你已经在代码中把 `base_url` 改为了 `https://www.qianjuai.com/v1`。不要漏掉“/v1”。

**错误2: `AuthenticationError: Incorrect API key provided`**
- **原因：** 你在千聚控制台生成的API密钥填错了，或者使用了OpenAI的原生密钥。
- **解决：** 检查 `api_key` 字段，复制的是千聚中转站控制台里的密钥，该密钥一般以 `sk-` 开头。

**错误3: `NotFoundError: The model 'grok-4-fast' does not exist`**
- **原因：** 模型名称写错了，比如用了`grok-4`（不带`-fast`）或者其他历史型号。
- **解决：** 请确认模型名称严格为 `grok-4-fast`。如果你想调用更高规格的版本，可查阅千聚官方文档中的支持模型列表，确认是否有其他型号。
