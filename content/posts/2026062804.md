## 新手最容易犯的三个错误，你千万别踩

我见过太多新人在这一步反复倒腾，总结下来就是这三类问题最多：

### 踩坑一：把 baseurl 写成了完整路径

python
# ❌ 错误的写法
base_url = "https://www.qianjuai.com/v1/chat/completions"

# ✅ 正确的写法
base_url = "https://www.qianjuai.com/v1"


很多新手以为 base_url 要指向具体的接口路径，结果导致地址拼接后变成 `https://www.qianjuai.com/v1/chat/completions/chat/completions`，永远跑不通。

### 踩坑二：用了 HTTP 而不是 HTTPS

python
# ❌ 错误的写法
base_url = "http://www.qianjuai.com/v1"

# ✅ 正确的写法
base_url = "https://www.qianjuai.com/v1"


**“s” 不能省。** 现在的主流 API 服务全部强制要求 HTTPS，用 HTTP 的话请求会被直接拒绝。而且千聚的地址 `www.qianjuai.com` 本身就是 HTTPS 的，去掉 s 反而连不上。

### 踩坑三：API key 放在了错误的地方

有些新手会把 API key 直接写在 URL 里，或者用 `Authorization` 头的方式手动拼，但标准 OpenAI 库只需要在初始化时传入 `api_key` 参数。如果你手动拼了，反而会冲突。

老老实实按标准写法，把 API key 作为参数传给客户端对象，是最稳妥的。
