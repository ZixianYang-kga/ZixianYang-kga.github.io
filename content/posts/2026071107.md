java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.util.ArrayList;
import java.util.List;

public class Gpt5ProCaller {

    public static void main(String[] args) {
        // 1. 设置千聚的API Key(替换成你自己的Key)
        String apiKey = "sk-你的千聚APIKey";
        
        // 2. 设置千聚的Base URL(核心改动点)
        String baseUrl = "https://www.qianjuai.com/v1";
        
        // 3. 创建OpenApiService实例,传入Key和自定义的Base URL
        OpenAiService service = new OpenAiService(apiKey);
        service.setBaseUrl(baseUrl); // 注意:如果你使用的SDK版本不同,设置方式可能略有差异

        // 4. 构建聊天请求(指定模型为GPT-5.2pro)
        List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage("system", "你是一位乐于助人的助手。"));
        messages.add(new ChatMessage("user", "用简单的语言解释什么是Java虚拟机(JVM)？"));

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gpt-5.2-pro") // 模型名称:GPT-5.2pro
                .messages(messages)
                .maxTokens(500) // 控制回复长度
                .build();

        // 5. 发送请求并打印回复
        System.out.println("正在调用GPT-5.2pro...");
        service.createChatCompletion(request).getChoices().forEach(choice -> {
            System.out.println("助手回复:");
            System.out.println(choice.getMessage().getContent());
        });
        
        // 6. 关闭service
        service.shutdownExecutor();
    }
}

## 遇到错误怎么办？排查指南

如果你运行代码时遇到错误,别慌,最常见的问题无外乎这几种:

| 错误信息                          | 原因                                        | 解决方案                                                               |
| --------------------------------- | ------------------------------------------- | ---------------------------------------------------------------------- |
| `401 Authentication Error`      | API Key 可能复制错了,或者Key已过期。        | 登录千聚控制台,重新创建一个新的API Key,并确保粘贴时没有多余空格。     |
| `404 Not Found`                 | `model` 参数填错了,或者 `base_url` 不对。   | 检查模型名称是否精确为 `gpt-5.2-pro`,检查 `base_url` 是否以 `/v1` 结尾。 |
| `429 Rate Limit Exceeded`       | 并发请求太多或短时间内请求过于频繁。        | 加入线程休眠,或者升级API Key的分组,增加并发限制。                      |
| `Connection Timeout`            | 网络不通或DNS解析失败。                     | 在命令行 `ping www.qianjuai.com` 测试连通性。如果不行,尝试切换一下网络。 |

如果排查了这些还是不行,千聚的官方文档里有一个“常见问题”板块,覆盖了大部分报错场景。另外,你可以在官网右下角找到客服入口,直接问他们。

## 从单次调用到批量处理：进阶用法

跑通一个单次请求只是第一步。在真实业务开发中,你往往需要批量或流式处理。这里给你两个进阶示例。

### 流式输出:` ` ` ` (Streaming)

适合聊天界面,像打字机一样一个字一个字地显示回复。

java
// 使用 jvm-openai SDK 的流式版本
ChatClient chatClient = openAI.chatClient();

CreateChatCompletionRequest request = CreateChatCompletionRequest.newBuilder()
    .model("gpt-5.2-pro")
    .addMessage(ChatMessage.user("写一首关于秋天的短诗"))
    .stream(true) // 开启流式输出
    .build();

chatClient.createChatCompletionStream(request)
    .forEach(chunk -> {
        chunk.choices().forEach(choice -> {
            if (choice.delta().content() != null) {
                System.out.print(choice.delta().content());
            }
        });
    });


### 函数调用(Function Calling)

让模型根据你的逻辑去执行一些外部操作,比如查询数据库或调用其他API。

java
// 示例:定义一个获取天气的函数,让模型决定是否调用它
List<FunctionDefinition> functions = List.of(
    FunctionDefinition.newBuilder()
        .name("get_weather")
        .description("获取指定城市的天气信息")
        .parameters(...) // 定义参数结构
        .build()
);

CreateChatCompletionRequest request = CreateChatCompletionRequest.newBuilder()
    .model("gpt-5.2-pro")
    .addMessage(ChatMessage.user("北京今天天气怎么样?"))
    .functions(functions) // 传入函数定义
    .build();


模型会返回一个 `tool_calls` 对象,告诉你要调用哪个函数,然后你在代码里执行并返回结果,模型再生成最终回复。这是构建智能体(Agent)应用的基础。
